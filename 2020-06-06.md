\01. 소셜커머스 : Social Commerce

<개념>

-커머스가 소셜미디어와 결합한 형태로 소셜미디어와 온라인 미디어를 활용하는 전자상거래의 일종

-소셜 네트워크 서비스(SNS)를 통하여 이루어지는 전자상거래

 

<부연설명>

-페이스북, 트위터 등의 소셜네트워크서비스(SNS)를 활용하여 이루어지는 전자상거래의 일종으로, 일정수 이상의 구매자가 모일 경우 파격적인 할인가로 상품을 제공하는 판매 방식

-소셜쇼핑이라고도함

-상품의 구매를 원하는 사람들이 할인을 성사시키기 위하여 공동구매자를 모으는 과정에서 주로 소셜네트워크서비스를 이용하기 때문에 이런 이름이 붙음

 

<유래>

-소셜커머스라는 용어는 2005년 야후의 장바구니 공유서비스인 쇼퍼스피어 같은 사이트를 통하여 처음 소개

-2008년 설립된 미굮의 온라인 할인쿠폰 업체 그루폰이 공동구매형 소셜커머스의 비즈니스 모델을 처음 만들어 성공을 거둔 이후 본격적으로 알려지시 시작함

-미국의 그루폰이 런칭한 후, 세계적으로 유행하고 있는 쇼핑 트렌드

 

<발전과정>

-최근 스마트폰 이용과 소셜 네트워크 서비스 이용이 대중화 되면서 새로운 소비 시장으로 주목받고 있음

-사람들과 사람들 사이의 사회적인 관계와 신뢰를 기반으로 특정한 서비스나 상품을 판매하는 것으로 확대

-예 : 야채 장수가 트위터로 자신의 이동경로를 알리거나, 페이스북에 상점을 여는 것 모두 소셜커머스로 볼 수 있음

-소셜커머스 업체가 등록한 상품은 단위 품목당 보통 24시간 동안 판매가 이루어지고, 대개 50%에서 90%까지의 높은 할인율이 적용됨. 단 일정수 이상이 구매해야 한다는 조건이 붙음

-주로 공연, 레스토랑, 까페, 미용 관련 소규모 사업자의 상품이 대량 판매되지만 레저, 패션, 가전제품, 식품 등의 상품들도 취급함

-판매업체가 박리다매와 홍보 효과를 기대하여 높은 할인율 적용

-일반적인 상품 판매는 광고와 마케팅 의존도가 높지만 소셜커머스는 소비자들이 소셜네트워크서비스를 통하여 자발적으로 상품을 홍보하면서 구매자를 모으기　 때문에 마케팅에 들어가는 비용이 거의 들지 않음

-그런 이유로 일부 업체는 소셜커머스 자체를 판매의 수단이 아니라 장기적인 고객을 확보하기 위한 홍보,마케팅의 수단으로 생각함

 

\02. 소셜 커머스의 4가지 유형

<소셜 링크형>

-커머스 사이트에 소셜 네트워크로 이동할 수 잇는 버튼 형식의 링크를 게재하는 방식

-버튼을 클릭하면 웹문서의 웹링크가 생성되어 해당 소셜 네트워크 글쓰기에 자동으로 삽입되거나, 자신의 SNS(트위터, 페이스북 등)에 글쓰기에 자동으로 삽입되거나, 자신의 SNS(트위터, 페이스북 등)dp 웹문서가 그대로 복사되어 게시물로 생성되게 하는 형태

-기존 장바구니와 유사

 

<소셜 웹형>
-커머스를 소셜 네트워크와 적극적으로 결합하는 것으로 커머스 사이트 안에서도 소셜 네트워크의 기능을 구현해 주는 방식

-커머스 사이트에서 이뤄지는 소비자의 구매, 평가 리뷰 등의 활동이 소비자의 소셜 네트워크에 자동으로 반영되어 친구들과 공유

-예 : SNS 플러그인 적용 / 게시물 작성하면 자동으로 트위터에 배포되는 기능

 

<공동구매형>

-공동구매 사이트가 소셜 네트워크와 결합한 형태

-소셜커머스 초기 가장 활발하게 활동하는 한 분야

-제품별로 정한 최소 구매 수량이 달서앟면 엄청난 할인 혜택을 받을 수 있도록 하여 소비자들로 하여금 적극적으로 소셜 네트워크를 통해 친구들을 공동구매에 참여시키고 있음

-초대한 친구가 회원가입을 하거나, 제품을 구매하면 현금 또는 포인트를 적립해주는 인센티브 프로그램 운영

-그루폰, 위폰 등이 대표적인 사례

 

<오프라인 연동>

-오프라인 공간을 네트워킹이 가능한 단말기로 소셜 네트워크와 연결시키는 유형

-위치기반 서비스를 활용하여 소비자의 오프라인 상점에서의 경험을 모바일로 소셜 네트워크에 확산시키는 방식과 매장에 비치한 컴퓨터로 바로 소비자의 소셜 네트워크에 접속할수 있도록 하는 유형

 

\03. 소셜 커머스 B2B 서비스 모델

<소셜 커머스 시장의 성숙에 따른 다양한 모델 등장>
-기존 커머스에 소셜네트워크를 연동해주는 웹 개발 서비스

-페이스북 등 소셜 네트워크 내에 쇼핑몰을 구축해 주는 서비스

-소셜 네트워크에 최적화된 상품 판매 정보 생성 솔루션

-소셜 네트워크에서 이용 가능한 결제솔루션

-자체적으로 공동 구매 이벤트를 진행할 수 있도록 도와주는 솔루션

-소셜 네트워크에서 실시간으로 홈쇼핑 방송을 할수 있도록 도와주는 솔루션

-상품 판매 정보를 소셜 네트워크 이외의 온라인 미디어에도 배포해 주는 네트워크 광고 서비스

-소셜 네트워크상의 입소문으로 매출 증대에 기여한 고객을 추적하고, 보상해 주는 솔루션 등

 

\04. 소셜커머스의 중요한 의미

<사이트의 트래픽을 증가시킬 수 있다>

-소비자가 친구를 초대하고 그 친구가 또다른 친구를 초대하는 과정을 통해 방문자가 늘어나게됨

 

<마케팅에 돈이 들지 않는다>

-친구추가 버튼을 누르면 상대방에게 친구요청이 전송됩니다. 상대방이 수락하면 친구로 맺어집니다.

 

1.이메일을 이용한 친구 찾기

-내비게이션 바 또는 사이드 바의 ‘친구 찾기’를 클릭하여 친구찾기 화면으로 이동합니다. 그런 다음 주로 사용하고 있는 메일 서비스를 선택하여 본인의 이메일 주소와 비밀 번호를 입력한뒤 <친구 찾기>를 클릭합니다.

-해당 이메일 주소에 저장된 주소록을 토대로 친구를 추천해 줍니다. 친구로 추가할 사람을 선택한 뒤 <친구 추가> 버튼을 클릭합니다.

 

2.프로필 정보를 기반으로 친구찾기

-내비게이션 바의 ‘홈’을 클릭한 뒤 사이드 바 ‘연결’ 항복에 있는 ‘친구 찾기’를 클릭합니다.

-프로필에 등록한 정보를 기준으로 친구를 찾으려면 해당 항목에 체크 표시를 합니다. 검색 결과가 나타나면 <친구 추가> 버튼을 클릭하여 친구 요청을 합니다.

-새로운 조건으로 검색하기 위해 앞에서 지정한 조건의 체크 표시를 없앱니다. 그런 다음 검색하고자 하는 항목의 입력란에 조건을 입력한 뒤 자동 완성 목록이 나타나면 원하는 항목을 선택합니다.

-두 가지 이상의 조건을 지정하면 모든 조건을 만족하는 검색 결과가 나타납니다. 검색 결과에서 <친구 추가> 버튼을 클릭하여 친구 요청을 합니다.

-내비게이션 바 또는 사이드 바의 ‘친구 찾기’는 주로 본인이 사용하는 이메일 계정의 주소록을 기반으로 메일을 주고 받은 사람을 검색해 페이스북 친구로 추천합니다.

 

3.이름으로 찾기

-내비게이션 바의 검색어 입력안에 찾고자 하는 사람의 이름을 입력합니다. 그런다음 결과 목록에 원하는 사람이 나타나면 클릭합니다. 만약 찾고자 하는 사람이 나타나지 않으면 목록 마지막에 있는 ‘에 대한 결과 더 보기’를 클릭합니다.

-사이드 바에서 ‘사람’을 클릭합니다. 그런 다음 ‘검색 도구’에서 <위치 ▼>를 클릭하여 검색하고자 하는 조건을 선택

-조건 입력란에 검색할 내용을 입력한 뒤 자동완성 목록이 나타나면 원하는 항목을 선택합니다.

-검색 결과에서 <친구 추가> 버튼을 클릭하여 친구 요청을 합니다.

-<친구 추가> 버튼을 클릭하는 즉시 친구 요청이 발송되며, 팝업 메뉴를 이용하여 해당 친구를 리스트에 포함하거나 친구 요청 취소 등의 작업을 할 수 있습니다

 

4.친구 수락하기

-나에게 친구 요청이 들어오면 내비게이션 바의 ‘친구 요청’ 아이콘에 빨간색 사각형으로 알림이 표시됩니다. 이를 클릭하면 친구 요청을 확인하 수 있는 창이 나타나며, <확인>을 클릭하여 친구요청을 승낙합니다.

-내비게이션 바의 내 이름을 클릭하여 프로필 타임라인으로 이동합니다. 그런 다음 프로필 정보의 ‘친구’를 클릭하거나 타임라인 상단에 위치한 친구의 ‘전체 보기’를 클릭하면 친구 목록이 나타납니다

5-3강 페이스북 활용하기

1.타임라인에 글 올리기

-내비게이션 바에서 내 이름을 클릭하여 프로필 타임라인으로 이동합니다. 그런 다음 상태 입력란에 내용을 입력합니다.

-글에 대한 공유 범위를 지정(여기에서는 ‘전체 공개’)한 뒤 <게시>를 클릭합니다.

-타임라인에 작성한 글이 나타납니다.

-내비게이션 바에서 ‘홈’을 클릭한 뒤 사이드바에서 ‘뉴스피드’를 선택합니다. 뉴스피드에도 내가 작성한 글이 나타납니다.

-뉴스피드에서 수정하기 : 공유 범위를 수정하고자 하는 글로 마우스 포인터를 이동합니다. 그러면 글 오른ㅉ고 위에 해당 글에 설정된 공유 범위 아이콘이 나타나며, 이를 클릭하여 공유 범위 수정 및 게시글 삭제 등의 명령을 실행할 수 있습니다.

-프로필 타임라인에서 수정하기 : 공유 범위를 수정하고자 하는 글 위쪽에 표시된 공유 범위 아이콘을 클릭하여 공유 범위를 수정합니다.

-활동 로그에서 수정하기 : 프로필 타임라인의 커버 스토리 아래에 있는 <활동 로그>를 클릭합니다. 활동 기록 중 글 오른쪽에 표시된 공유 범위 아이콘을 클릭하여 공유 범위를 수정합니다.

 

2.친구 타임라인에 글 남기기

-내비게이션 바에서 내 이름을 클릭하여 프로필 타임라인으로 이동합니다. 그런 다음 타임라인 위쪽에 있는 친구 항목에서 친구의 프로필 사진 또는 이름을 클릭합니다.

*친구가 많아 원하는 친구의 프로필 사진이 나타나지 않을 경우 프로필 타임라인의 친구 항목에 있는 ‘전체 보기’를 클릭하거나 프로필 정보의 ‘친구’를 클릭하면 친구 전체 목록을 확인할 수 있습니다.

-글 입력란에 내용을 입력한뒤 <공유하기>를 클릭합니다

-친구의 타임라인에 내가 쓴 글이 올라갑니다

-친구의 커버 스토리 아래에 있는 버튼을 클릭한 뒤 [친구 사이 보기]를 선택합니다

-나와 친구 사이에 오간 글이나 댓글 등을 일목요연하게 확인할 수 있습니다

-친구가 내 타임라인에 글을 올린 경우 내비게이션 바의 알림 아이콘에 빨간색 숫자가 표시됩니다. 알림 아이콘을 클릭하여 해당 알림을 선택합니다.

-화면이 바뀌면서 친구가 남긴 글을 확인할 수 있으며, 내비게이션바의 내 이름을 클릭하여 타임라인에서 확인할 수도 있습니다.

*뉴스피드 또는 타임라인의 글 앞에 표시된 프로필 사진을 클릭하여 해당 친구의 타임라인에 방문할 수도 있습니다.

*친구가 내 타임라인에 글을 올리는 것을 원하지 않을 경우

 ①내비게이션 바의 ▼를 클릭한 뒤 ‘공개범위설정’을 클릭합니다
 ②공개범위 설정화면이 나타나면 ‘교류방법’에 해당하는 ‘설정관리’를 클릭합니다

 

 

 

 

3.사진 올리기

-타임라인의 상태 입력란 위에 있는 ‘사진’을 클릭합니다

-‘사진/동영상 업로드’를 클릭합니다. 그런 다음 <파일 선택>을 클릭하여 원하는 이미지를 선택합니다. 입력란에 내용을 입력합니다.

-글 아래에 있는 시계 아이콘을 클릭하여 게시글을 올리고자 하는 연도, 월, 일을 차례로 지정합니다. 그런 다음 공유 범위를 지정한 뒤 <게시>를 클릭합니다

-타임라인의 해당 날짜 위치에 글이 게시됩니다. 사진을 클릭합니다.

-사진 창이 나타나면 해당 사진에 대해 ‘다운로드’, ‘프로필 사진으로 지정’, ‘사진 삭제’ 등 의 명령을 지정할 수 있습니다. <X>를 클릭하여 사진창을 닫습니다

-퀵메뉴의 내 이름 버튼을 클릭하거나 내비게이션바의 내 이름을 클릭하여 타임라인 최상단으로 화면을 이동합니다

-여러 장의 사진을 올리기 위해 ‘사진’을 클릭한뒤 ‘사진첩 만들기’를 선택합니다. 그런 다음 [열기]대화상자가 나타나면 업로드 하고자 하는 사진을 <Ctrl> 누른 상태에서 클릭하여 선택한 뒤 <열기>를 클릭합니다.

-사진첩 편집 화면이 나타나면 사진첩 제목 및 각 사진에 대한 설명을 입력합니다.

-사진첩 편집 화면 오른쪽 위에 있는 ‘어디서 찍은 사진인가요?’에 지명을 입력한 뒤 자동완성 목록이 나타나면 원하는 항목을 선택합니다

-‘날짜 추가’를 클릭하여 연도, 월, 일을 지정한 뒤 <저장>을 클릭합니다

-사진에서 태그를 달고자 하는 영역을 클릭합니다. 그런 다음 태그를 달 친구 이름을 입력한 뒤 자동 완성 목록이 나타나면 친구를 선택합니다.

-태그가 지정된 사진 아래의 사람 모양 아이콘을 클릭하면 사진에 태그된 친구 이름이 나타납니다. 공유 범위를 지정한 뒤 <사진 올리기>를 클릭합니다

-사진첩 완성 화면이 나타나면 내비게이션 바의 내 이름을 클릭하여 프로필 타임라인으로 이동합니다

-프로필 히스토리에서 사진첩 날짜에 해당하는 기간을 선택하여 해당 시간대로 타임라인의 위치를 이동합니다

-선택한 시간대로 타임라인이 이동하면 앞에서 올린 사진첩이 나타나며, 이때 사진첩의 사진 중 하나를 클릭합니다. 사진 창이 나타나면 좌우의 ◀,▶를 클릭하여 사진첩의 다른 사진을 봅니다. 친구가 태그된 사진의 경우 아래에 태그된 친구 이름이 나타나며, 친구 이름에 마우스 포인터를 이동하면 사진에서 태그된 부분이 표시됩니다.

*내가 태그된 사진보기

 방법1 - ‘알림’ 아이콘을 클릭하여 해당 소식을 선택합니다, 친구에 의해 내가 태그된 사진을 바로 확인할 수가 있습니다.

 방법2 - 타임라인의 프로필 정보에 있는 ‘사진’을 클릭합니다, ‘회원님의 사진’에서 내가 태그된 사진을 확인할 수 있습니다.

-다른 메뉴를 선택하기 위해 화면 왼쪽 위에 있는 메뉴 버튼을 터치합니다. 그런 다음 메뉴가 나타나면 친구의 타임라인으로 가기 위해 ‘친구’를 터치합니다

-친구 목록이 나타나면 원하는 친구를 터치합니다. 그러면 친구의 타임라인이 나타납니다

 

3.페이스북 앱에서 글쓰기

\01. 스토리텔링 특징

<스토리텔링 개념>

-스토리(story)란?

-인물, 사건, 배경을 조합하여 만들어 낸 한 덩어리의 말하기

-누가(인물) 언제 어디에서(배경) 어떻게 하였다(사건) 라는 내용을 갖춤

 

-스토리텔링(story telling)이란?

-인물, 사건, 배경을 갖추어 어떤 내용을 전달하는 스토리를 표현하는 말하기 행위

 

<스토리텔링 성격>

-storytelling=story + tell + ing

-story : 상황적 이해 가능, 관계성, 개연성, 보편성

-tell : 상호작용, 소통

-ing : 현장성, 몰입도 강화

-storytelling : 현실적, 감성적, 놀이적, 관계적 성격

 

<삶과 스토리텔링

-스토리텔링특징

-인지력 : 스토리텔링은 비선형적이며 유연하고 다양한 의미를 농축

-현실성 : 현실의문제, 현재적 삶의 보편적 가치 수렴

-상호텍스트 : 다양한매체, 다양한 시각의 수용 가능성 중시

-이야기의 힘은 이야기가 우리의 삶에 밀착되어 있음을 보여준다.

-인간은 끈임없이 이야기하는 존재. 나는 이야기하므로 존재한다.

-스토리텔링이 우리에게 주는 것은? 삶에 대한 관계적 이해와 재미

-기호의 변화 : 문자중심사회 -> 영상중심사회

-매체의 변화 : 인쇄출판 -> 다양한 매체로(게임기, DMB, 스마트폰 등 모바일 기기, 온라인 도구 새로운 매체 등장

-방식의 변화 : 이성중심 -> 감성중심

-목적의 변화 : 지식적 속성 -> 놀이적 속성

-감각정 영상매체 문화콘텐츠의 핵심은 스토리텔링

 

-네트워크사회로의이동

-관계성, 현장성, 상호작용성

-현실적 관계 작용을 보여줄 수 있는 유일한 방식은 스토리텔링

 

-글로벌사회

-상업적, 무국적성

-국경, 민족을 초월하는 문화사업의 흥미적 힘은 스토리텔링

-두개이상의 스토리텔링을 결합시켜 다중적인 메시지도 가능

 

-상호작용이 뛰어나다

-서로간의 소통을 유도하면서 상호작용을 하도록 함

 

-고정된 구조가 아니다

-언제나 유동적이므로 교환 가능함

-새로운 이야기로 변주할 수 있고, 다른 이야기와 섞일 수도 있음

\01. UCC(User Created Contents)

<일반적 정의>

-사용자가 직접 제작한 콘텐츠 -> 온라인(웹)상에 업로드 하는 것

-사용자가 상업적인 의도 없이 제작한 콘텐츠

-동영상(Video), 글(Text), 이미지(Image), 음성(Audio) 등을 포함

 

<국가별 개념 차이>

-한국 -> UCC(User Created Contents) : 사용자 제작 콘텐츠

-미국 -> UGC(User Generated Contents) : 사용자 생성 콘텐츠

-일본 -> CGM(Consumer Generated Media) : 고객 발신형 미디어

 

\02. UCC 관련 용어 이해하기

<유튜브(YouTube)>

-사용자가 비디오 클립을 업로드하고, 동영상을 보거나 공유

-YouTube(미국), 유튜브 코리아, 유튜브 일본, 유튜브 모바일 등

 

<퍼블리즌(Publizen)=Publicity(공개, 홍보) + Citizen(시민)>

-인터넷을 통해 자신을 대중에게 적극적으로 노출하는 사람들

-자기 의지로 자신의 장기나 표현하고 싶은 모습 등을 동영상으로 촬영해 인터넷에 공개하는 사람들을 지칭(인기와 유명세?)

 

<프로슈머(Prosumer)=Producer(생산자)+Consumer(소비자)>

-전문가 못지 않는 지식과 관심을 갖고 기업의 경제활동에 막대한 영향력을 발휘하는 소비자들을 지칭

-인터넷 안의 모든 콘텐츠와 문화는 사실상 사용자들이 만들어 나가고 소비하는 새로운 수요와 트렌드를 반영하는 용어

 

플래쉬몹(Flash mob)

-불특정 다수가 인터넷, 이메일, 휴대폰을 통해 정해진 시간, 장소에 집결한 뒤 모버레이터(moberator : 지시서를 나눠주는 사람)들이 나눠준 지시서에 따라 특정행동을 하고 순식간에 사라지는 행위를 지칭하는 신조어

-플래쉬몹은 혼자놀기에 익숙한 네티즌들이 고안한 새로운 종류의 유희로서 놀이와 매체의 결합에서 생겨난 문화

-그 형태는 촛불시위부터 시체놀이까지 규정지을 수 없을 만큼 매우 다양

-최근에는 한류 열풍을 타고 K-POP 플래쉬몹이 대 유행

②Narrative(Story Telling)(이야기형식)

-이야기 형식에 따라 내용을 전개하는 방법

③Problem/Solutior(문제 해결)

-문제 해결 방식, 문제를 먼저 언급하고 그에 따른 해결책을 제시하면서 구성하는 방법

④Cause/Effect(원인과 결과)

-원인과 결과, 원인과 그에 따른 결과를 설명하는 방법으로 아웃라인을 잡는 방법

⑤Topical(주제별)

-주제별, 일반적인 주제를 여러 개의 세부 주제로 나눠서 제시하는 방식

⑥Journalistic questions(육하원칙에 따라)

-육하원칙에 따라 구성하는 방법, 즉 누가, 무엇을, 언제, 어디서, 어떻게, 왜 라는 질문에 따라 내용을 전개하는 방법

⑦Spatical(공간의)

-방향, 위치, 공간을 기준으로 내용을 구성하는 방식

③Large - 그림과 문자는 크게 표현

-여러 장의 슬라이드로 나누어 한 슬라이드에 하나씩 크게

④Visual - 언어 논리의 시각화 표현(키워드→구조화→시각화)

-시각화된 자료의 우수한 정보 전달성(주의집중 및 강한 이상 부여)

-이해의 용이성 및 짧은 시간 내 많은 정보의 전달성

-사각형, 원, 화살표와 같은 기본 도형으로 표현(적절한 템플릿 사용)

⑤Graphic - 수치는 맞춤형 그래프로 구현

-데이터에서 결정할 수 있는 핵심 메시지 결정

-메세지가 직접 드러나는 제목 결정(ex : "4년간 매출액 두 배 증가”)

-구성요소 : 각 부분의 크기를 전체의 백분율로 나타낸 것(원형)

-항목 : 대상의 순위를 비교(가로막대)

-시간적 추이 : 대상이 시간에 따라 어떻게 변화하는지 관심(세로막대, 선그래프)

-도수분포 : 얼마나 많은 항목이 일련의 점진적 수치 범위 내에 해당하는가를 표현

-상관관계 : 변수 사이의 관계가 기대하는 패턴을 따르는지 여부

⑥Legible - 가독성 높은 글꼴, 색상 등으로 디자인

⑦Less - 양은 적당하게/적게

-직접적으로 연결되는 핵심요소의 중요 메시지만 전달

\1. 시선처리

①프레젠터가 가장 먼저 시선을 두어야 할 부분: 맨 뒷부분

-맨 뒷부분에 있는 사람을 보고 이야기 하면 자기 목소리가 뒤에 있는 사람까지 잘 전달되는지 가늠할 수 있음

-맨 뒤에 앉는 분들이 보통 핵심적인 인물이기 때문에 시선을 먼저 둠(key man을 중심으로 시선처리)

②시선을 자연스럽게 옮겨다니면서 1:1 eye contact

③프레젠터의 의견을 잘 들어주고 집중해 주는 사람에게 두 번째로 시선을 둠

-잘 쳐다보지 않고, 산만하거나 관심이 없는 사람에게 시선을 주다보면 프레젠터 스스로가 자신감이 떨어짐

-자신에게 우호적인 사람들에게 자주 eye contact → 심리적인 자신감 확보

④마지막으로 모두에게 골고루 시선을 줌

-1:1 eye contact

⑤효과적인 청중과의 상호작용을 통해 효율적인 시선처리가 이루어져야 함

-프레젠터 자신감 확보 및 성공적인 프레젠테이션 유도

 

⑥시선처리 일반원칙

-시선처리는 자료와 도구를 보는 비율과 청중을 보는 비율을 50:50으로 하는 것이 원칙

-의사결정권자에게 우선적으로 시선을 주되, 지나친 치중 불가

-자신에게 우호적인 청중들에게 골고루 시선 처리 유지

②마무리 잘하기 위한 원칙

-내용을 다시 요약/정리해 주고, 재동기를 부여하고 압축함

-다른 부분보다 더 강한 인상을 주어야 함(화면은 더 인상깊게, 강한 어조, 마무리로서의 임팩트)

-자기 주장이 가미된 결론 메시지가 분명해야 함

-단순 지식전달 외에 의사결정을 유도하기 위한 분명한 논조 어필

-새로운 정보를 언급하지 않는다

-부정적인 어조 또는 상투적인 말로 끝내지 않는다

-다 끝나기 전에 자료를 챙기지 않는다

-감성적인 마무리 멘트를 활용하라

 

\9. 질의 응답 스킬 업

①청중이 곤란하거나, 당황스러운 질문을 해서 프레젠터가 그 질문에 명쾌하게 답변을 못하거나, 질의응답에 소홀하게 되면 프레젠테이션은 결과적으로 실패하게 됨

②“질문할 경우” 원칙

-청중에게의 가벼운 질문 : 환기효과/동기부여

-청중에게 골고루 질문함

-청중의 수준을 고려함

-자문자답은 금물

③“질문 받을 경우” 매끄러운 질의응답 원칙

1)질문하는 사람에게 시선을 둔다

-질문자 칭찬, 자연스럽게 존중하는 마음으로 eye contact

-질문을 하고 있는 도중에 프레젠터가 다른 곳을 보거나, 다른 일을 하게 되면 질문하는 청중은 무시당하는 느낌이 들 수있음

-항상 프레젠터는 청중의 입장에서 청중을 존중해 주는 마음 표시

2)애매하거나, 장황한 질문은 적절하게 조정

-다른 청중들을 고려하여 질문 정리 후, 답변

예)예, 무슨 말씀인지 알겠습니다. 바로 이런 이런 말씀이시군요~

-프레젠터는 융통성 있게 질문에 대한 통제와 조정을 해야 함

3)질문 내용은 청중 전체가 공유하도록 한다.

4)프레젠테이션 진행 도중 질문이 나왔을 경우에는 프레젠테이션이 모두 끝난 후, 질의응답을 하도록 유도한다.

④“질문에 답할 경우” 원칙

1)질문을 받으면 한 박자 정도 쉬고 대답한다.

-곧바로 대답할 경우 논리적이지 못할 수 있다.

2)처중의 질문에 대해 반박하는 질문을 하지 않는다.

-예)그러면 본인은 어떻게 생각하십니까?(X)

3)어떤 질문이라도 답변의 의무를 성실히 한다.

-당장 답변하기 곤란할 경우 차후에 답변해 줄 것을 약속한다.

-무표정한 얼굴과 단조로운 목소리는 지양해야 한다.

\2. 효과적인 커뮤니케이션 원리

①상대가 알아듣기 쉬운 말로 하라.

-쉽게 설명하지 않으면 상대는 이해하지 못할 수도 있다.

-미사여구, 거창한 말 보다는 귀에 쏙 들어오는 말로 해야 한다.

-비즈니스에서 중요한 요소가 된 “단순화”가 바로 핵심 컨셉이다.

-군더더기 없는 말로 표현하라.

-이해하기 쉬운 말로 표현하는 것이 대화의 예의이다.

②열정적으로 말하라.

-열정적으로 말하면 상대방도 감동하여 열정적으로 반응한다.

-어려운 상황에서도 미래의 비전을 보여주는 긍정의 메시지를 전달한다.

③일체감을 느낄 때 커뮤니케이션이 잘 이루어진다.

-사람과 소통할　때 동료 의식을 강조하면 실패하는 경우는 드물다.

-‘나’라는 말보다 ‘우리’라는 말을 강조한다.

④시각화의 원칙

-사람들은 눈에 보이는 것은 믿는다.

-자신에 대한 좋은 이미지를 만들어야 한다.

-결국, 이미지 메이킹이 중요하다.

⑤유머의 원칙

-유머는 커뮤니케이션의 2% 부족한 부분을 채워 준다.

-이 시대의 코드는 재치와 유머이다.

-유머가 중요한 경쟁력이다.

-유머를 사용할　때는 타이밍과 장소를 잘 선택할 줄 아는 지혜가 필요하다.

⑥간결성의 원칙

-한줄로 된 짧은 메시지가 강한 인상을 각인시키는 데에 효과적이다.

-사람과 소통할 때 메셎가 단순하고 간결해야 한다.

-짧고 핵심적인 말은 상대에 강한 인상을 심어준다. 

 

\3. 올바른 커뮤니케이션 자세

①예의를 지킨다.

-예의를 갖추어 자신의 의견을 자연스럽게 전달할 줄 아는 사람이 인간관계에서 성공한다.

-인사는 중요한 자기 표현의 하나이다.

-고마운 사람에게는 “고맙습니다”, “감사합니다”라고 고마움의 표현을 해야한다.

②분명한 어조로 말한다.

-분명한 어조는 강한 인상을 심어준다

-자신의 의견을 당당하게 말할 줄 알아야 한다.

-비판을 할 때에는 바드시 대안을 제시해야 한다.

-상대를 무시하거나, 도전적인 어조, 예의 없는 말투는 지양해야 한다.

-사람에 대한 신뢰를 바탕으로 자신의 뜻을 분명한 어조로 말해야 한다.

③쓸데없는 말을 하지 않는다.

-핵심을 벗어난 쓸데없는 말은 본심을 숨기려 하거나, 과대 포장하는 행위이다.

-쓸데없는 말을 하지 않는다는 것은 그만큼 자신과 상대를 믿는다는 뜻이다.

④재치가 있다.

-재치는 상대나 상황에 얽매이지 않아야 제대로 발휘할 수 있다.

-그때그때 상황에 따라서 재빨리 발휘되는 ‘지혜’, ‘기지’이다.

-재치 있는 사람일수록 우리는 깊은 대화를 나눌 수 있다.

⑤남 헐뜯기를 좋아하지 않는다.

-남을 헐뜯는 것은 열등감이 많기 때문이다.

-내 앞에서 남을 헐뜯게 되면 결국 신뢰감을 잃게 된다.

-커뮤니케이션을 잘하는 사람은 남을 헐뜯지 않는다.

-남의 장점을 적극적으로 찾아내어 관계를 돈독하게 해야 한다.

-인간관계에서 굳이 적을 만들지 말아야 한다.

⑥말의 사용방법에 주의한다.

-내가 쓰는 말은 나의 인격을 나타낸다.

-대화를 할　때에는 어떤 말을, 어떻게 사용할 것인가를 충분히 생각해야 한다.

-커뮤니케이션에 능숙한 사람은 언어의 선택과 말하는 태도에 특별히 주의를 기울인다.

⑦무례한 언어는 사용하지 않는다.

-같은 뜻의 말이라도 억양에 따라서 다르게 받아들여진다.

-커뮤니케이션에 능숙한 사람은 상대가 좋게 받아들일수 있는 말을 사용한다.

-욕, 무시, 결함, 비방 등은 지양해야 한다.

⑧사람마다의 대화 스타일을 정확하게 파악한다.

-상대방의 말투에서 그 사람의 화법을 판단하라.

-같은 말이라도 사람마다 말의 강도가 다르다는 것을 명심해야 한다.

-상대방이 애매모호한 표현을 할　때는 그냥 넘어가지 말고 되물어 보는 것이 좋다.

-사람마다 대화의 스타일은 천차만별! 상대방의 말버릇과 습관을 파악해 말 속에 담긴 정확한 의도를 파악해야 한다.

①새로운 형태의 원천 데이터가 기하급수적으로 새엉

-기존에는 데이터로서 생성이 필요 없던 데이터가 이제는 데이터로 생성되기 때문

-새로운 목적과 서비스를 위해 데이터들이 생성

-(예) 위치 기반 정보가 군사용에서 내비게이션, 대중교통시스템 등에 활용

②디지털 데이터의 폭발적 증가

-기존 아날로그 데이터의 디지털 데이터화

-단순 디지털 파일들 특히 비정형화되어 있는 데이터들의 정형화화 구조화

③디지털 데이터를 증가하게 하는 채널의 다양화와 새로운 채널의 출현

-디지털 데이터를 만드는 다양한 디바이스 증가

-소셜네트워크와 같은 인터넷 서비스로 인해 데이터 증가

④기업 환경의 글로벌화와 기업 환경의 변화

-글로벌화가 가속화되고 성장과 경쟁이 치열

-다양한 분야에서 기업이 필요로 하는 정보를 찾고 분석하며 새로운 상품과 서비스를 개발하고 연구해야 하는 환경

⑤마케팅 업무에서 데이터 분석이 점점 중요해지고 있는 현실

-자사의 마케팅에 관한 효과와 실제 투자 대비 성과 분석 중요

-마케팅 업무에서 데이터 분석이 매우 중요

-마케팅 데이터는 구조화되지 않거나 텍스트와 구문 등 비정형 데이터가 대부분이기에 분석 기술도 계속 발달

⑥고객의 빠른 대응

-신제품 출시 후에 고객의 반응이 매우 즉각적으로 피드백

-고객 반응 분석과 자동화는 상당히 중요한 부분

⑦소셜네터워크의 특징인 빠른 전달력

-실시간화 되는 고객의 반응에 즉각 대응, 실시간으로 데이터 수집 또는 추출

-실시간으로 흘러가는 비정형 불규칙 데이터를 빠르게 추출하고 수집하여 분석

⑧대용량 데이터의 축적과 빠른 데이터 처리

-다양한 포맷의 커뮤니케이션은 대용량 데이터 전송과 저장소가 필요

⑨글로벌 경제 확대

-글로벌화에 따른 신용카드 이용 정보의 급증, 엄청난 데이터의 저장소 필요

\01. 빅데이터, 어떤 특징이 있어야 하는가?

<빅데이터 세 가지 특징>

①규모(Volume)

-데이터의 크기

-축적되는 데이터의 크기가 수직 확장성의 물리적 한계를 초과하는 거대한 데이터 크기를 의미

-단일 데이터로서 볼륨이 큰 데이터가 있을 것이며, 대규모의 데이터 집합체도 이에 해당

-MB, GB 단위 규모 → TB, PB, EB 단위

-미국 기업의 37% 이상이 이미 10TB 이상의 빅데이터 보유

②다양성(Variety)

-데이터의 다양성으로 다양한 형식과 채널

-다양한 데이터 형식의 통합은 결국 고비용을 발생시키므로 저비용 고효율 처리가 관건

-정형 데이터 : 특정 형식에 맞춰 잘 정리된 데이터 (예) 기업에서 다루는 재무데이터들

-비정형 데이터 : 데이터 하나하나마다 크기와 내용이 사뭇 달라 통일된 구조로 정리하기 어려운 비정형, 비구조적 데이터 (예) 동영상, 음악, 메시지, 소셜미디어, 위치정보, 게시물 등

③속도(Velocity)

-데이터 변화와 축적, 분석 속도를 의미

-데이터의 변화 속도보다 더 빠른 결정이 이루어지는 것을 의미

-빅데이터 시대의 기업에는 사건 발생과 데이터 수집 사이의 간격을 좁혀 더욱 역동적으로 경영 환경 변화에 대응할 기회 확대

-과거에는 기업 활동에 대한 반응을 수집하고 이를 새로운 의사결정에 반영하려면 짧아도 며칠, 길면 몇 주 이상 소요 → 그 주기가 몇 시간, 심지어 분 단위로 단축

-(예) 트위터상에서 ‘빈 라덴 사망’ 소식의 전파 모습 : 초당 5,000회

 

\02. 빅데이터의 좁은 의미와 넓은 의미

<세 차원을 종합한 빅데이터 정의 - (좁은 의미의) 빅데이터!

-보통 수십에서 수천 테라바이트 정도의 거대한 크기를 갖고, 여러 가지 다양한 비정형 데이터를 포함하고 있으며, 생성-유통-소비(이용)가 몇 초에서 몇시간 단위로 일어나 기존의 방식으로는 관리와 분석이 매우 어려운 데이터 ㅈㅂ합을 의미

 

<(넓은 의미의) 빅데이터!>

①빅데이터는 과거에 주로 다룬 비교적 작은 크기의 정형화된 데이터에서 쓰이던 것과는 다른 차원의 기술과 인력이 요구

②좁은 의미의 빅데이터를 관리하고 분석하는데 필요한 인력과 조직, 제반 기술과 인프라를 따로 떼어놓고 생각하기는 어려움

③빅데이터란 기존의 방식으로 관리와 분석이 매우 어려운 데이터 집합, 그리고 이를 관리⋅분석이 매우 어려운 데이터 집합, 그리고 이를 관리⋅분석하기 위해 필요한 인력과 조직 및 관련 기술까지 포괄하는 용어

 

<빅데이터! 좁은 의미 Vs 넓은 의미>

-좁은 의미의 빅데이터 : 정형 데이터, 비정형 데이터 + 데이터 관리⋅분석 인력과 조직 + 데이터 간리⋅분석 관련기술

 

<기업 빅데이터의 두 가지 유형>

①빅거래 데이터

-구조적 형태의 데이터로서 각종 거래를 처리하는 과정에서 발생하는 데이터

-거래는 특정한

②빅상호작용 데이터

-비구조적 데이터로서 사람들간, 사람과 기계간, 혹은 기계들과의 상호작용으로 생산되는 데이터

-사람이 구사하는 언어와 기계 간에 주고받는 기초적인 정보는 제각각의 체계화되지 않은 데이터가 대부분

 

<얼마나 커야 ‘빅데이터’인가?>

①데이터 단위와 크기

-킬로바이트(KB) 1000< 메가바이트(MB) 1,000,000< 페타바이트(PB)< 엑사바이트(EB)< 제타바이트(ZB)< 요타바이트(YB)

②테라바이트(Tb) : 1,024GB

-20만장의 사진 또는 MP3 노래를 담을 수 있는 하드 드라이브 하나 정도의 크기

-일반적인 컴퓨터에서 사용되는 단위의 저장 공간 크기

③1페타바이트(PB) : 1,024TB

-16개의 스토리지팟 정도의 규모

-데이터센터의 랙 2개에 해당

-웬만한 인터넷 기반 서비스는 기본 단위가 페타바이트 정도

④1엑사바이트(EB) :1,024PB

-1PB짜리 랙을 2천 개를 모아 4층 규모의 데이터 센터 빌딩에 가득 채운 상태

⑤1제타바이트(ZB) : 1,024EB

-데이터 센터를 1천개 합치 규모로 뉴욕 맨하탄의 20%에 해당

⑥1요타바이트(YB) :1,024ZB

-수백만개의 데이터 센터를 합해놓을 정도로 매우 큰 규모의 데이터 단위

 

\03. 빅데이터의 데이터 유형 분류

<구조화(정형화)된 데이터>

①Structured Data

②관계형 데이터베이스에서 데이터 형식이 정규화되어 있고 데이터 스키마가 지원

-데이터 간에 연계성을 바탕으로 다양한 데이터 조합을 만들어서 데이터 정렬과 분석을 쉽고 빠르게 할 수 있는 장점 보유

<반-구조화 데이터>

①Semi-Structured Data

②다소 정형화된 파일이 데이터 속성인 메타데이터를 가지며 자체적으로 데이터에 관한 서술이 가능한 형태

③데이터베이스 형태는 아니지만 XML 형태로 술되고 구조화되어 있는 데이터

<준-구조화 데이터>

①Quasi-Structured Data

②다소 일관성이 없는 데이터의 형식

③일반적으로 웹사이트에서 사용자에 의해 발생하는 클릭스트림데이터가 이에 해당

④클릭스트림데이터는 웹사이트에서 발생하는 사용자의 다양한 경험 데이터

<비구조화 데이터>

①Non-Structured Data

②일반 텍스트 기반의 데이터로 분석이 가능한 데이터

③텍스트화되지 않는 데이터로는 이미지 파일과 동영상과 같은 멀티미디어 데이터

②기존 데이터베이스 소프트웨어 시장이나 스토리지 시장에 많은 변화가 예상

②표준화 된 하드웨어 상에서 쉽게 확장되는 아키텍처 필요

③빅데이터 기술과 솔루션에 부합되는 IT 인프라

 

<신규 데이터 소스>

①데이터가 입수되는 신규 채널

②새로운 타입의 원천 데이터들을 다루게 될 것으로 예상

③센서, 디바이스, 기존 애플리케이션, 웹서버, 공공데이터 등

 

<경제환경 변화>

②데이터 기반의 혁신 비즈니스가 가속화

③새로운 비즈니스와 다양한 업종의 비즈니스, 인공지능에 가까운 비즈니스 서비스 등장

②실시간 정보 활용

③실시간 의사결정 지원을 위한 수 초내 응답 가능한 쿼리 적용

 

<다이내믹 데이터>

②패턴 기반의 알고리즘이 고도화되어 복잡하면서도 대규모로 확장 가능한 성능은 더 빠르게 진화하는 모델로 성장

 

\03. 빅데이터 ‘무엇이’ 문제이고. ‘왜’ 문제인가?

<빅데이터 집합 및 관리의 문제>

①각 시스템 및 데이터 베이스별로 분리된 조직에 따라 데이터에 대한 소유권이 나뉘어 있어서 데이터를 통합하고 수집하는 초기부터 프로젝트 진행의 어려움 발생

-통합된 TF팀 구성 후, 데이터 활용에 관한 권한 부여

②기존의 데이터 분석 체계와 빅데이터의 큰 차이 발생

③빅데이터로 분류해야 하는 데이터의 인식에 관한 차이

④데이터의 대부분인 비정형 데이터의 활용성 미흡
